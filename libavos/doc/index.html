LibAvos is a library of functions that can be used in C programs, to write stand-alone applications for the Archos JBMM (JukeBox MultiMedia) and the Archos AV3x0 platforms. This document talks you through the process of setting up a cross-compilation environment that will allow you to use LibAvos in your programs. If you follow the steps outlined here, it should be very easy to get yourself set up. This document assumes that you are running some form of Unix system that supports the GNU tool-chain.

The first step is to download and install binutils. When you have downloaded the tar.gz, unpack it by typing

tar -xzf binutils-*.tar.gz

Now move into the binutils directory and build and install binutils with

./configure --target=arm-elf
make
su # go root
make install
exit # return from root

Next, download and unpack gcc. Move into the gcc directory and build and install gcc by typing

./configure --target=arm-elf --disable-threads --enable-languages=c
make
su # go root
make install
exit # return from root

Now get LibAvos from avos.sf.net. Move into the libavos directory and create a symbolic link from hardware.S to one of jbmm.S or av300.S. For example, if you have an Archos AV340, type

ln -s av300.S hardware.S

Now you can build LibAvos by typing

make clean
make
su # go root
make install
exit # return from root

You will then most likely want to build the example "Hello World" program, which you can do by typing

make hello

You will, if all has gone well, be left with a hello.bin file, that you can pack up (using the packer that comes with RockBox) and put on your Archos. To write your own C programs you will need to include libavos.h and link against /usr/local/lib/libavos (consult the Makefile for details of how this is done). As well as the "Hello World" program, we have included a palette program which you can compile easily by modifying the Makefile (good practice for when you want to do Makefiles for your own programs!).
