HOW-TO build the arm-linux Toolchains ?

Answer: Try to follow the hereunder guide!



These guide is based on this page.

These guide has been made for Linux users, but it should be usable in Cygwin with slight modification.



  1. Let's create a specific directory in home for all work related to linAV.

    cd ~
    mkdir linav
    cd linav

  2. We get the sources from CVS.

    cvs -d:pserver:anonymous@uclibc.org:/var/cvs login
    cvs -z3 -d:pserver:anonymous@uclibc.org:/var/cvs co -P toolchain

    After the first line, you will be prompted for a password. Simply press Return.

  3. Some modification needed in the Makefile. download this file and put it in ~/linav/toolchain/gcc-3.3.x

    cd toolchain/gcc-3.3.x


    With your favorite text editor do the following changes:
    Comment the line:     #ARCH:=i386
    Uncomment the line: ARCH:=arm
    Change the line: SOFT_FLOAT:=false to SOFT_FLOAT:=true
    Save your change. it should be enough.

  4. The make processus will download some file and then ask you some questions:
    - First question: 1- Generic Arm
    - Second question: n – no mmu
    - Third question : n – no debug

  5. A long time of compilation... Be patient !

  6. Some links are missing (don't know why !)

    cd toolchain_arm_nofpu/arm-linux/lib/

    ln -s ../../lib/gcc-lib/arm-linux-uclibc/3.3.*/libgcc.a

    ln -s ../../lib/libfloat.a


  1. It should be ok now . Let's test these toolchain by compiling avos.

    su your_user

    cd ~/linav

    cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/avos login

    cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/avos co avos2

  2. We add The toolchains to the default path.

    export PATH=$PATH:~/linav/toolchain/gcc-3.3.x/toolchain_arm_nofpu/bin/
    cat “export PATH=$PATH:~/linav/toolchain/gcc-3.3.x/toolchain_arm_nofpu/bin/” >> .bashrc
    cd avos2


  1. Edit the Makefile.conf in order to have the three first line as hereunder:


    #ARM toolchain root directory
    BUILDDIR = ~/linav/toolchain/gcc-3.3.x/toolchain_arm_nofpu
    BUILDVER = 3.3.3

  2. Let's test!

    make


  1. It should be OK!
